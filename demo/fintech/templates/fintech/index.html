Fintech Audit Logging

Endpoints

Exercise the demo endpoints below to generate audit events, then inspect logs/sec-audit.jsonl for the OTel JSONL output (HTTP responses, auth signals, model changes, scrubbed request bodies).

  • GET /accounts/ — list accounts (model access events)
  • GET /accounts/<id>/ — account detail
  • POST /transfers/ — create a transfer
  • POST /transfers/high-risk/ — route to risk review
  • POST /auth/login/ — demo login (auth signal)
  • POST /profile/update/ — update profile (scrubbed body capture)
  • GET /demo/generate-events/ — generate synthetic traffic
