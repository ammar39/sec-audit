# Sec Audit Monitoring

This directory was generated by `sec-audit-loki-init`. It tails
django-sec-audit OpenTelemetry LogRecord-shaped JSONL with Grafana Alloy and
pushes logs to Loki for Grafana dashboards. Stdout collection is the
recommended production path; this generated file setup is a secondary local
example.

## Expected Django Logging

Configure Django to write the audit JSONL file that Alloy mounts:

```python
AUDIT_LOG_PATH = 'logs/sec-audit.jsonl'

SEC_AUDIT = {
    'logging': {'schema_version': '1.0'},
}

LOGGING = {
    'version': 1,
    'formatters': {
        'audit_jsonl': {
            '()': 'sec_audit.django.logging.formatters.audit_jsonl_formatter',
        },
    },
    'handlers': {
        # Alloy tails this JSONL file; a stdlib RotatingFileHandler with the
        # canonical formatter writes it. (Stdout is the supported prod path.)
        'audit_file': {
            'class': 'logging.handlers.RotatingFileHandler',
            'filename': AUDIT_LOG_PATH,
            'maxBytes': 10 * 1024 * 1024,
            'backupCount': 3,
            'formatter': 'audit_jsonl',
        },
    },
    'loggers': {
        'sec_audit.audit': {
            'handlers': ['audit_file'],
            'level': 'INFO',
            'propagate': False,
        },
    },
}
```

The generated Docker Compose file mounts:

```text
{{ AUDIT_LOG_PATH }} -> /var/log/sec-audit/sec-audit.jsonl
```

Make sure the host path exists before starting the stack.

## Run Locally

From your project root:

```bash
mkdir -p logs
touch logs/sec-audit.jsonl
docker compose -f monitoring/docker-compose.yml up -d
```

Then generate at least one audited request in your Django app.

## Verify

Check Alloy loaded the config and started tailing the audit file:

```bash
docker compose -f monitoring/docker-compose.yml logs alloy --tail=30
```

Check Loki received events:

```bash
curl -G http://127.0.0.1:3100/loki/api/v1/query \
  --data-urlencode 'query=count_over_time({service_name="{{ APP_LABEL }}"}[5m])'
```

Grafana runs at `http://localhost:3000`. Log in with the credentials rendered in
`docker-compose.yml` and open the provisioned Sec Audit dashboard.

## Labels And Queries

Alloy sets these Loki stream labels:

- `service_name="{{ APP_LABEL }}"`
- `environment="{{ ENVIRONMENT }}"`
- `severity`, copied from `severity_text`
- `event_type`, copied from `attributes.event_type`

Keep high-cardinality fields such as `source.address`, `url.path`, `user.id`,
and `session_id` in the JSON body. Query them with the `| json` LogQL stage.
See `loki/queries.md` for examples.

## Regenerate

When upgrading django-sec-audit templates:

```bash
sec-audit-loki-init monitoring --overwrite \
  --audit-log-path {{ AUDIT_LOG_PATH }} \
  --app-label {{ APP_LABEL }} \
  --environment {{ ENVIRONMENT }}
```
